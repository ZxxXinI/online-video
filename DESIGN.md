# 在线影院设计系统

## Design Read

“在线影院”是高频浏览型影视工具，不是营销落地页。这一版设计以 `Spotify` 的 content-first darkness 为主参考，借一点 `PlayStation` 的娱乐化陈列感：界面退到暗处，让海报、片名和播放入口成为真正的前景内容。

## Design Dials

- `DESIGN_VARIANCE=5`：允许首页有更强的英雄区和影院氛围，但不做花哨装饰。
- `MOTION_INTENSITY=2`：只保留轻微抬升、缩放和焦点环。
- `VISUAL_DENSITY=7`：桌面六列、手机三列，信息密度保持高效浏览。

## Tokens

- Canvas：默认深色，主画布约为 `#090b0e`，营造影院暗场。
- Surface：分为 `surface`、`surface-raised`、`surface-subtle` 三层，依靠层级和轻边框分组。
- Accent：主强调色为功能型绿色 `#1ed760`，用于主要 CTA、选中态和可操作焦点。
- Accent Secondary：蓝色 `#53b1ff` 仅用于细小高光和渐变收尾，不主导页面。
- Radius：`8px / 14px / 24px`，列表和卡片偏圆润，但不做营销式大泡泡。
- Type：使用系统中英文字体；首页与详情标题重且大，辅助说明与元数据保持克制。

## Component Rules

- 侧栏导航是内容入口，不做后台菜单感；当前项需要有底色、图标高亮和明确焦点。
- 搜索框、主题切换、Tab、分页、选集项必须拥有 hover、focus-visible 和 selected 状态。
- 海报保持 `2:3` 比例；片名最多两行；右下角状态标签使用半透明胶囊。
- 首页必须先给内容气氛，再给分类列表；英雄区允许真实海报做背景，但文字必须可读。
- 详情页和播放页沿用同一套暗色层级，不允许突然切回明亮工具面板。

## Do

- 让页面看起来像“选片”和“看片”的地方。
- 用真正的海报、片名、分区标题建立层次。
- 在移动端保留大部分品牌氛围，但保证搜索和分类先可用。

## Do Not

- 不使用炫技型渐变按钮、玻璃泡泡大卡片、装饰性图标墙。
- 不让工具控件压过内容本身。
- 不在首页塞过多说明文字；说明只做辅助，不抢主视觉。

## Implementation Notes

- 当前实现基于 Tailwind 和语义 CSS variables。
- 保留明暗主题切换，但默认仍偏向深色影院模式。
- 首页 hero 背景图使用普通 `img`，因为外部海报域名动态且不适合静态优化代理。
